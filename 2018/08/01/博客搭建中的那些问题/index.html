博客搭建中的那些问题

发表于 2018-08-01

我的博客由Hexo作为博客框架，Github提供的300M免费空间作为服务器，Next作为博客的主题。这个博客真的是搭建不易啊，在这儿我就数数我掉到几个坑里去了。。。

##1.创建仓库##
到这个页面去创建一个新仓库 http://https://github.com/new
这个新仓库就将是存放你即将拥有的博客的地方
注意，你的仓库名不能随便取，这样会导致github混乱，取名的格式应该为“用户名.github.io”

emmm，我就在这儿掉进去了一次，因为用户名和github的用户名不一样，所以出现了错误，但是也是可是弥补的，那就是删除了这个仓库重新建，那么要怎么删除了这个仓库的呢。。
下面说一下，要怎么删除仓库。

  1. 首先选中你要删除的仓库，比如我要删除这个dhw614714/python这个仓库；
  2. 打开这个仓库之后看见上面的setting，打开它；
  3. 打开之后一直拉到最下面，这时你会看到一个Delete this repository,点击它然后再确定你要删除的仓库即可。

##2.Hexo本地测试的时候要注意##

  1. Git要运行在本地Hexo的根目录下；
  2. Git中Ctrl+C是stop的快捷键，不要使用Ctrl + C或Ctrl + V的快捷键，用右键；
  3. 本地测试是Hexo is running,Git显示如下，其他状态便无法连接；

##3.一些需要注意的细节##

  1. 网上的搭建博客步骤不一定全部都适合你，所以你在搭建的过程一定要注意自己出现的问题及时解决；
  2. 我们在搭建主题的时候，会需要将主题设定，在next主题目录下的_config.yml文件中将scheme设定为Pisces
    注意一定要记得加空格，不然后面会报错，在后面设置主题中会遇到很多添加的，有的一定要看清楚是否要加空格；
  3. 语言设定，在站点根目录下修改配置文件_cofig.yml中的language为zh-Hans（简体中文）要进入到主题目录下的languages文件中修改zh-Hans.yml，但是当我去我的languages中占我的zh-Hans.yml，就是找不到，我的是zh-CN.yml，我把它删了，删了，，然后重新找别人的复制了一个zh-Hans.yml，然后就一直有错误，这个要记住不能删，这个也是相当于zh-Hans.yml，可以在这儿里面改；
  4. 我们在主题的增改中，一定要记得修改一点部署一下，以便检查是否有什么错误，后来改正找的太辛苦。。
  5. 部署可以通过这三步
  • hexo d
  • hexo g
  • hexo s 生成加预览
