第二关

题目链接

这关就是循环的尝试密码，需要用到requests的post请求。
最常见的是以from形式发送post请求，requests提交Form表单,一般存在于网站的登录，用来提交用户名和密码。，只需要将请求的参数构造成一个字典，然后传给requests.post()的data参数即可。

url = 'http://www.heibanke.com/lesson/crawler_ex01/'
d = {'key1': 'value1', 'key2': 'value2'}
r = requests.post(url, data=d)
print r.text

程序如下：

#!/usr/bin/env python
# -- coding: utf-8 --
import requests
from bs4 import BeautifulSoup
import urllib.request
url= 'http://www.heibanke.com/lesson/crawler_ex01/'
wrong = '您输入的密码错误, 请重新输入'
index = 1
while True:
    data = {'username':'admin','password':index}
    html = requests.post(url,data).text
    print (html)
    if wrong not in html:
        print (index)
        break
    print ('第%d次访问，密码错误'%index)
    index += 1

思路是这样的：因为密码是1-30中的一个数，所以我们我们可以通过一个循环更改密码然后使用requests的post请求将我们的用户名和密码提交，我们观察页面代码知道，如果密码错误的话，它会给我们’您输入的密码错误, 请重新输入’这句话，所以我们可以通过这个来写循环。

题目链接
这关就比较复杂点了，我们来到这关的时候看到了这里是黑板客爬虫闯关的第三关，比上一关多了两层保护的字样
看到顶部有个注销按钮，应该先登录才可以做题，，emmmmm，这个我是自己想不到的，右键查看元素->网络，然后再输入用户名密码登录，看右边面板中的参数，也就是我们发送的数据：
，我们看到了除了发送用户名和密码，还发送了一个csrfmiddlewaretoken的数据，观察发现题目的页面，刷新下网页，在Response Headers也就是服务器响应的数据的Set-Cookie里发现有个csrftoken的字段，它和我们登陆后的表单数据里的csrfmiddlewaretoken是一样的。在这两个登录页面是都有csrfmiddlewaretoken这个参数的，所以这就应该是那两次“保护”。
程序如下：
import requests
from bs4 import BeautifulSoup

url_1='http://www.heibanke.com/accounts/login'
url_2='http://www.heibanke.com/lesson/crawler_ex02'

data_1={}#手动创建的用户和密码
data_1['username']='dhw614'
data_1['password']=614714
data_1['csrfmiddlewaretoken']=''


data_2={}#试探的用户和密码
data_2['username']='dhw'
data_2['csrfmiddlewaretoken']=''
data_2['password']=0

s=requests.Session()#存储特定用户会话所需的信息。这样，当用户在应用程序的 Web 页之间跳转时，存储在 Session 对象中的变量将不会丢失，而是在整个用户会话中一直存在下去

s.get(url_1)
data_1['csrfmiddlewaretoken']=s.cookies['csrftoken']#通过发链接获得csrfmiddlewaretoken的值
s.post(url_1,data_1)#发送数据
data_2['csrfmiddlewaretoken']=s.cookies['csrftoken']#得到另外一个csrfmiddlewaretoken，csrfmiddlewaretoken不是一直不变的，每次访问值都不同

for i in range(1,31):#密码猜测
    data_2['password']=str(i)
    result=s.post("http://www.heibanke.com/lesson/crawler_ex02/",data=data_2)
    if result.text.find('错误') == -1:
        print('密码是：'+data_2['password'])
        break
    else:
        print(data_2['password']+'不是密码')
        continue

  1. find（）函数
    描述：Python find() 方法检测字符串中是否包含子字符串 str ，如果指定 beg（开始） 和 end（结束） 范围，则检查是否包含在指定范围内，如果包含子字符串返回开始的索引值，否则返回-1。
    语法：str.find(str, beg=0, end=len(string))
  • str – 指定检索的字符串
  • beg – 开始索引，默认为0。
  • end – 结束索引，默认为字符串的长度
    返回值：如果包含子字符串返回开始的索引值，否则返回-1。

2.find_all
语法：find_all( name , attrs , recursive , string , **kwargs )
find_all() 方法搜索当前tag的所有tag子节点,并判断是否符合过滤器的条件。

  • name参数：这是最简单而直接的一种办法了，可以通过html标签明来索引；

    soup.find_all("a", string="Elsie")
